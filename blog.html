25 Things I learned at Portland Code School this week (in no particular order)

It was a packed week of learning ... lots of HTML/CSS review and Ruby of course, but lots of broader information for the first week. Here's a quick snapshot:

  1. PATFT
  2. Using SRT files for Treehouse for review shortcut
  3. How to play Go
  4. Tim Berners Lee and the Semantic Web; his dream was for computers to talk to computers; we're not quite there yet
  5. There are a plethora of text/code editors available (i.e. VIM, emacs, Sublime, Textmate, etc.) and that the goal of a programmer is to keep your hands off the mouse as much as possible.
  6. An ongoing war among programmers: spaces versus tabs. Ruby styleguide states "Use soft-tabs with a two space indent."
  7. Extra spaces and the end of lines of code are collectively called "meow"?
  8. Mac Terminal executes BASH which stands for "born again shell"
  9. There are three types of structured data markup: microdata (Google prefers), microformats, and RFDa
  10. In Ruby, blocks are NOT objects. Savable blocks are called Procs.
  11. API stands for "Application Programming Interface". The terms is generally used in two different contexts. When one writes to the API, he/she is either referring to the contract/agreement or the actual server.
  12. CSS normalize, which is different from CSS reset by Eric Meyer
  13. HTTP #418 is "I'm a teapot"
  14. Font-squirrel another source for free fonts
  15. Some apps that watch a specific directory for updates and automatically reload view in browser: live reload, active reload (javascript), and watcher (ruby)
  16. Different pair coding styles ... heavy driver, doer versus research assistant, one person speaks the other types
  17. Use no more than 3 slides per 5 minute lighting talk. MUST rehearse!
  18. It's possible to quit the internet :)
  19. Enumerable is a module of array class
  20. The term "falsiness" comes from Stephen Colbert
  21. MTBEA = "meantime between expression and evaluation"
  22. "Duck typing" (if it walks and quacks it must be a duck) versus "duck punching" (punch object into the shape of a duck)
  23. Everything in Ruby is both and advantage and a curse.
  24. Ruby methods: inject, reduce, reject, map (all use each)
  25. "Yak shaving" the best term ever
